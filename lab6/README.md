# Лабораторная работа №6: Тестирование REST-сервисов в SoapUI

## Описание бизнес-процесса

Бизнес-процесс: **Получение текущей погоды в столице заданной страны.**

Сценарий состоит из двух последовательных шагов (запросов к публичным REST API):
1. **Шаг 1: Получение информации о стране (REST Countries API)**
   - **Запрос**: `GET https://restcountries.com/v3.1/name/{country}`
   - **Цель**: Получить координаты (широту и долготу) столицы этой страны из JSON-ответа.
2. **Шаг 2: Получение погоды по координатам (Open-Meteo API)**
   - **Запрос**: `GET https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}&current_weather=true`
   - **Цель**: Узнать текущую температуру в столице, используя координаты, полученные на предыдущем шаге.

---

## Структура проекта

1. **`lab6-soapui-project.xml`** — файл проекта для SoapUI. Содержит:
   - REST-сервисы с параметрами.
   - TestSuite с именем `WeatherTestSuite`.
   - TestCase с именем `GetWeatherWorkflow`.
   - Шаги теста:
     - `GetCountryInfo` (REST Request)
     - `CoordinateTransfer` (Property Transfer — переносит широту и долготу из ответа первого шага в параметры второго).
     - `GetWeather` (REST Request)
   - Проверки (Assertions):
     - Статус-коды ответов (HTTP 200).
     - Наличие корректных данных в JSON.
2. **`workflow.js`** — простой JavaScript-скрипт для быстрой проверки этого сценария через консоль (работает на встроенном `fetch` в Node.js).

---

## Как запустить SoapUI проект

1. Откройте SoapUI.
2. Выберите **File** -> **Import Project** и выберите файл `lab6-soapui-project.xml`.
3. Разверните дерево проекта, найдите TestCase `GetWeatherWorkflow` и запустите его (кнопка плей).
4. Все шаги должны выполниться успешно (зеленый статус).

---

## Быстрый запуск в консоли

Выполните команду в папке `lab6`:
```bash
node workflow.js
```
Скрипт выполнит те же самые шаги и выведет температуру в столице.
