# Отчет по лабораторной работе №7: WebAssembly

## Тема работы
Изучение, компиляция и доработка веб-приложения Budget Planner (Планировщик бюджета) на базе WebAssembly.

## Изучение архитектуры проекта
Проект демонстрирует интеграцию низкоуровневого кода на C с браузерным JavaScript:
* `main.c` — содержит структуры для расходов (`ExpenseEntry`) и подсчета итогов (`CategoryTotal`). Здесь реализованы все основные вычисления, которые работают в бинарном формате в браузере.
* `app.js` — отвечает за обработку действий пользователя в интерфейсе, выделение памяти в Wasm для передачи текстовых строк и чтение результатов.
* `index.html` — пользовательский интерфейс для управления расходами.
* `makefile` — конфигурация для компиляции проекта через Emscripten.

---

## Установка Emscripten SDK
Для сборки проекта я установил компилятор Emscripten:
1. Склонировал репозиторий SDK:
   ```bash
   git clone https://github.com/emscripten-core/emsdk.git emsdk
   ```
2. Установил актуальные инструменты:
   ```bash
   ./emsdk/emsdk install latest
   ```
3. Активировал компилятор:
   ```bash
   ./emsdk/emsdk activate latest
   ```
4. Добавил пути в текущее окружение терминала:
   ```bash
   source emsdk/emsdk_env.sh
   ```

## Доработки проекта (Улучшения)

Я внес несколько ключевых изменений в проект:
1. **Контроль лимита бюджета**:
   * В код на C добавлены переменная лимита и функции `jsSetBudgetLimit` и `jsGetRemainingBudget`.
   * При превышении лимита расходов остаток бюджета автоматически подсвечивается красным цветом.
2. **Экспорт расходов в файл CSV (Собственное улучшение)**:
   * На стороне C реализовал функцию `jsExportCSV()`, которая собирает все внесенные расходы в одну строку формата CSV.
   * На стороне JS написал обработчик, который считывает эту строку из памяти Wasm, преобразует ее в обычную JS-строку, скачивает файл `expenses.csv` на компьютер пользователя и освобождает занятую Wasm-память.
   * В HTML добавил кнопку «Экспортировать в CSV».
3. **Локализация**:
   * Полностью перевел интерфейс приложения (`index.html`) и комментарии к коду в файлах `main.c` и `app.js` на русский язык, чтобы проект выглядел аккуратно.

## Сборка и запуск
1. Сборка Wasm-модуля:
   ```bash
   make
   ```
2. Запуск локального сервера для обхода ограничений CORS при загрузке `.wasm` файла:
   ```bash
   python3 -m http.server 8000
   ```
3. Открыть в браузере: `http://localhost:8000/`. Приложение корректно работает, считает бюджет и скачивает CSV-файл с расходами.
